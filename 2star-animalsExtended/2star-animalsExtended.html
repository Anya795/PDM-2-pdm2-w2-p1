Instructions

This exercise builds on the 1 star exercise, so you will need to complete that exercise before beginning this one.

Your task is to extend the Animal class you wrote in the previous exercise with new functionality described below. Start by copying the code and assets from the last exercise.

New Functionality

Some animals are dangerous, some are not. Your Animal class should have a way to represent the dangerousness of an individual animal. You will definitely need a new attribute but you should decide:

  • Should it be public or private? OOP best practice is to make all attributes private, unless there's a good reason why users of the class should be able to freely edit its value.
  • What values should it have? The sample solution uses three integers to represent not dangerous, may be dangerous, definitely dangerous, but feel free to represent this attribute in another way.
  • How should the dangerousness be set? Should it be set at creation via the constructor or via a method? Should it be possible to change?

Lastly, modify the display() method to create a visual representation of danger. The video below uses a simple coloured border. Feel free to be more creative!

There are no tests for this exercise as it is open ended.

Optional extension: consider using exceptions to ensure that code using your class sets dangerousness appropriately.
