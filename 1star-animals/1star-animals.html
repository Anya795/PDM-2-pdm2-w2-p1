Your sketch:

Instructions

In this exercise, you will create and use a class following the specification below. Writing code to a specification means you have been given the exact inputs and outputs expected, including the names of all public parts of the classâ€”it's name, constructor parameters, and method names with arguments and return types. Writing to specification can be a little bit easier than designing classes yourself but it becomes very important that you follow the specification exactly. Following specifications is an important aspect of one popular approach to software development, Test Driven Development (TDD), in which automated tests of a program are written before the program itself.

Implement the class

Write a class that meets the following specification:

  • Class name: Animal
  • Constructor parameters: x (number), y (number), image (p5.js Image). The last parameter should be an image returned by p5.js's loadImage() function.
  • Public method: display() (no parameters or return). Draws the animal's image at its x and y coordinates.
  • Public method: moveX(deltaX). Modifies the animal's x coordinate by deltaX (a number).
  • Public method: moveY(deltaY). Modifies the animal's y coordinate by deltaY (a number).

You may add additional attributes or methods if you like but they must be private to avoid breaking the specification. Hint: you will need private attributes to store the values passed to the contructor.

Run the check file for this exercise. It will test whether your class implementation matches the specification. You may need to comment out any incomplete starter code in order to run the tests.

Use the class

Complete the rest of the sketch as follows:

  • Create three different Animal objects with different images and place them at different locations on the canvas. You may use images downloaded from the internet or the images provided in the assets folder. The starter code includes one Animal (although the call to the constructor is missing arguments). You may reuse this object or start over from scratch.
  • In draw(), display all animals using their display() method.
  • Complete the implementation of keyPressed() to move each animal using the WASD keys and the moveX and moveY methods.

There are no tests for this part of the exercise.

Expected output:

Your class will need a private attribute for each argument passed to the constructor. Remember that to make an attribute private, use the # symbol before its name (with no gap between # and the name).

When you implement the constructor and other methods, you will need to call (i.e. use) the class attributes. Remember that to use an attribute or method inside the class, you need to use this. before the name. For example, if you have an attribute #x, you can call it in your class by typing this.#x.

Before creating instances of your class in the second part of the exercise, you will need to load all images you want to use in the preload() function. If you need a reminder of this step, take a look at the loadImage() documentation. You can create your Animal objects in preload() if you like. If you want to create them in setup(), you will need to store the loaded images somewhere.

If you can't remember how to draw an image in p5.js, see the image() documentation.
